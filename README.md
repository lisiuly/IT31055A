# IT31055A 室内温湿度监测仪固件项目

## 项目简介

IT31055A 是一个基于 Generalplus GPL813 平台开发的室内温湿度监测仪固件项目，面向带 LCD 显示的消费类环境监测产品。项目实现了温湿度采集、趋势显示、历史记录、霉菌报警、低电检测、按键交互、背光与蜂鸣器控制等完整业务链路。

这是一个真实产品固件项目，同时也具备很强的简历展示价值，因为它覆盖了：

- 传感器采集与软件 I2C 通信
- LCD 显示驱动与图标状态控制
- 多页面 UI 交互与按键状态机
- 历史记录、趋势算法、告警逻辑等业务处理
- 中断、定时节拍、休眠唤醒等底层系统设计

## 项目亮点

### 1. 完整的温湿度产品功能闭环

- 支持室内温度显示，范围为 `-9.9 C ~ +50.0 C`
- 支持室内湿度显示，范围为 `1%RH ~ 99%RH`
- 支持 `C/F` 单位切换
- 支持 Today / 48hr / All-time 历史记录展示
- 支持温湿度趋势图标显示
- 支持霉菌报警阈值设置与蜂鸣器报警
- 支持低电检测与背光控制

### 2. 业务逻辑较完整，接近真实量产产品

项目不是单纯的传感器 Demo，而是完整覆盖了消费类仪表常见的产品逻辑，包括：

- 超量程显示 `HH/LL`
- 历史记录清除与恢复展示
- 多历史页面切换
- 霉菌阈值设置页
- 趋势平缓/上升/下降状态切换
- 低电状态显示
- 休眠与唤醒流程

### 3. 兼顾底层驱动与上层业务

项目既涉及底层硬件控制，也涉及较复杂的上层数据处理：

- 底层：GPIO、软件 I2C、定时中断、蜂鸣器、背光、睡眠唤醒
- 上层：历史记录结构设计、趋势判断算法、页面显示逻辑、按键交互逻辑

## 负责内容

围绕这个真实产品项目，README 当前重点梳理了可用于项目经历表达的技术工作：

### 1. 温湿度历史记录链路梳理与修正

- 重构并梳理 Today / 48hr / All-time 历史记录的数据流
- 修正历史记录偏移和 max/min 更新问题
- 处理温度 `HH/LL` 时对历史记录链路的影响
- 将温湿度历史有效位拆分，避免温度异常时把湿度历史一并清空

### 2. 趋势算法优化

- 将趋势判断由简单累计逻辑改为窗口法
- 支持温度跨度超过 `1.0 C`、湿度跨度超过 `5%RH` 时触发趋势变化
- 当 1 小时内未超阈值时显示平缓趋势
- 上电后初始化趋势显示为平缓，避免界面空白

### 3. 显示与异常边界处理

- 处理温度超量程时的 `HH/LL` 显示逻辑
- 处理湿度显示上限与历史记录一致性问题
- 修正清除历史记录后 `--.-` / `--%` 的显示反馈
- 保证显示层与数据层状态一致，避免页面展示错乱

### 4. 告警与交互逻辑调整

- 梳理蜂鸣器、按键音、霉菌报警的状态关系
- 调整报警中按键音抑制逻辑
- 处理背光、蜂鸣器与按键逻辑之间的配合
- 结合中断节拍保证交互体验和显示刷新

## 技术实现

### 软件架构

项目主要模块如下：

- [DEMO/Startup.asm](DEMO/Startup.asm)：上电、主循环、休眠与唤醒
- [DEMO/Int_Vec.asm](DEMO/Int_Vec.asm)：中断入口与节拍处理
- [DEMO/Alarm/Alarm.asm](DEMO/Alarm/Alarm.asm)：温湿度数据处理、历史记录、趋势、低电、霉菌报警
- [DEMO/LCD/LCD_Display.asm](DEMO/LCD/LCD_Display.asm)：LCD 数值与图标显示
- [DEMO/KEY/KEY.asm](DEMO/KEY/KEY.asm)：按键扫描、页面切换、背光、按键音
- [DEMO/GXHTV4/GXHTV4.asm](DEMO/GXHTV4/GXHTV4.asm)：传感器读数与换算
- [DEMO/I2C/D_I2C.asm](DEMO/I2C/D_I2C.asm)：软件 I2C 驱动

### 硬件接口

主要 I/O 映射如下：

- `PB0`：背光
- `PB1`：蜂鸣器
- `PE0`：F/C/CLEAR 键
- `PE1`：ALL-TIME 键
- `PE2`：LIGHT 键
- `PD3`：I2C SCL
- `PD4`：I2C SDA
