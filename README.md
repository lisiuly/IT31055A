# IT31055A 室内温湿度监测仪固件工程

## 1. 项目定位

IT31055A 是一个真实产品固件工程，对应一款带 LCD 显示的室内温湿度监测仪。工程基于 Generalplus GPL813 平台开发，核心能力包括温湿度采集、LCD 显示、历史记录、趋势指示、霉菌报警、低电检测、按键交互、背光和蜂鸣器控制。

- 产品功能维护与固件迭代
- 规格书对照验证与仿真调试
- 显示、历史记录、趋势和告警逻辑排查

## 2. 规格梳理

### 2.1 基本功能

- 3 个按键：`F/C/CLEAR`、`ALL-TIME`、`LIGHT`
- 支持温度单位 `C/F` 切换
- 支持背光
- 支持低电图标显示
- 支持霉菌报警阈值设置，默认值 `65%`

### 2.2 测量与显示范围

- 室内温度显示范围：`-9.9 C ~ +50.0 C`
- 室内华氏显示范围：`+14.0 F ~ +122 F`
- 室内湿度显示范围：`1%RH ~ 99%RH`
- 温度分辨率：`0.1`
- 湿度分辨率：`1%RH`
- 采样周期：`60 秒`

### 2.3 页面与交互逻辑

- 标准模式下，短按 `F/C/CLEAR` 切换温度单位
- 标准模式下，长按 `F/C/CLEAR` 3 秒清除当前历史记录
- 标准模式下，短按 `ALL-TIME` 切换 `Today / 48hr / All-time` 历史视图
- 标准模式下，长按 `ALL-TIME` 3 秒进入 Mold 报警阈值设置
- 标准模式下，短按 `LIGHT` 点亮背光 8 秒

### 2.4 告警与状态显示

- 温度低于量程下限时显示 `LL.L`
- 温度高于量程上限时显示 `HH.H`
- 湿度超出舒适区时显示干燥 / 舒适 / 潮湿状态
- 湿度超过 Mold 阈值时显示 Mold 图标并触发蜂鸣器报警
- 工作电压低于 `2.5V` 时显示低电符号

### 2.5 历史与趋势

- 支持 `Today / 48hr / All-time` 温湿度历史记录
- 长按清除当前历史记录后，显示 `--.-` 和 `--%` 约 3 秒
- 温度累计变化超过 `1.0 C` 显示升降趋势
- 湿度累计变化超过 `5%RH` 显示升降趋势
- 1 小时内变化未超阈值时显示平缓趋势

## 4. 关键实现要点

结合当前代码，工程中比较关键的实现点包括：

### 4.1 历史记录设计

- History 采用 record 结构存储温度最大值、最小值、华氏值和湿度最大最小值
- `48hr` 页面不是单独完整存一套，而是由 `48HrToday + Previous Day` 聚合得到
- 清除历史记录时，只清当前页对应的数据，不影响其他历史页
- 当前温度 `HH/LL` 与湿度历史有效性已拆分处理，避免温度异常带坏湿度历史显示

### 4.2 趋势判断逻辑

- 趋势已经按窗口法处理，不再只是简单累加
- 温度窗口跨度超过 `1.0 C` 触发升降趋势
- 湿度窗口跨度超过 `5%RH` 触发升降趋势
- 若 1 小时内未超阈值，则显示平缓趋势
- 上电后趋势初始化为平缓，避免图标空白

### 4.3 异常边界处理

- 温度超量程显示 `HH/LL`
- 湿度显示和历史记录限制在两位范围内
- 历史清除后先显示 dash，再恢复当前刷新数据
- 显示层与数据层通过兼容缓冲衔接，LCD 继续复用旧 max/min 显示链路

### 4.4 告警与交互

- Mold 报警由湿度阈值驱动，并结合蜂鸣器和图标显示
- 按键音、报警音、背光都依赖统一节拍和状态位配合
- 主循环与中断分工明确：界面和业务在主循环，中断负责节拍和蜂鸣器波形
