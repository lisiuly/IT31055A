# 历史 48hr / All-time 与主功能调试测试流程

## 1. 先说结论

当前代码里的关键行为如下：

1. 历史采样不是每次刷新都记，而是每 60 秒记一次。
2. 触发条件是 RTC 秒数等于 00，并且命中半秒边沿一次。
3. 标准页本身就是 Today 页。
4. 短按 ALL-TIME 的页面行为是：标准页进入 48hr，之后在 48hr 和 All-time 之间切换。
5. 历史页不会靠短按 ALL-TIME 回 Today，而是 20 秒无操作后自动回标准页。
6. 48hr 不是单独一套完整存储，而是 48HrToday 和 Previous Day 两个桶聚合显示。
7. 长按 F/C/CLEAR 3 秒清当前页历史。
8. 清除范围分别是：标准页清 Today，48hr 页清 48HrToday 和 Previous Day，All-time 页清 All-time。
9. 清除后前台会先显示约 3 秒横杠，后台会立刻用当前样本重建当前页对应的历史桶。

对应代码落点：

1. 采样入口：Alarm/Alarm.asm 里的 F_JudgeRFC、F_UpdateTHFromGXHTV4、F_GetTHVlaue、F_HistoryOnSample。
2. 页面和按键：KEY/KEY.asm 里的 Enable_AllTimeKey、Enable_FcClearKey、Enable_longFcClearKey、Enable_longAllTimeKey。
3. 历史页签和图标显示：LCD/LCD_Display.asm 里的 Disp_ProductShowHistoryTab。

## 2. 推荐调试方法

建议分两阶段做：

1. 先做软件注数，验证页面切换、历史桶、清除范围、趋势和图标逻辑。
2. 再做实机环境测试，验证真实传感器、低电、背光和蜂鸣器。

原因很直接：

1. 当前历史采样是 60 秒周期，靠手工吹气、冰喷、热风枪来做 48hr 和 all-time 的逻辑验证，速度太慢也不稳定。
2. 48hr 还依赖跨日行为，不注数的话很难快速复现。

## 3. 软件注数怎么做

### 3.1 最小改动注数点

最适合注数的位置是 Alarm/Alarm.asm 的 F_UpdateTHFromGXHTV4。

优点：

1. 可以跳过真实 I2C 读传感器。
2. 后面的温度换算、湿度换算、HH/LL、历史记录、趋势、Mold 图标都会照常走。
3. 便于快速替换成一组固定测试向量。

临时注数示例：

```asm
F_UpdateTHFromGXHTV4:
    ; 临时测试注数，测试后恢复原始读传感器代码
    LDA     #00H
    STA     R_TempCH
    LDA     #0FDH
    STA     R_TempCL
    LDA     #2DH
    STA     R_HUM
    JMP     CalculateRFC
```

上面这组数据表示：25.3C，45%RH。

### 3.2 数据格式

软件注数时，直接写这 3 个变量：

1. R_TempCH
2. R_TempCL
3. R_HUM

格式如下：

1. 温度是 0.1C 单位的双字节值。
2. R_TempCH 的 bit7 是负号位，1 表示负温，0 表示正温。
3. 正温时，温度值就是无符号 16 位整数，例如 25.3C 就是 253，也就是 00FDH。
4. 负温时，绝对值照样按 0.1C 填，最后把 R_TempCH 的 bit7 置 1，例如 -5.6C 就是 8038H。
5. 湿度 R_HUM 直接填整数百分比，范围 0 到 100。

## 4. 推荐测试向量

为了减少边界歧义，尽量用明显超过阈值的点，不要卡在临界点上。

| 场景 | 目标值 | R_TempCH | R_TempCL | R_HUM | 预期现象 |
| --- | --- | --- | --- | --- | --- |
| 基线样本 A | 20.0C / 40% | 00H | C8H | 28H | 正常显示，无 HH/LL |
| 基线样本 B | 30.0C / 60% | 01H | 2CH | 3CH | 正常显示，适合做 max/min |
| 基线样本 C | 25.0C / 50% | 00H | FAH | 32H | 适合做清除后重建样本 |
| 负温正常 | -5.6C / 80% | 80H | 38H | 50H | 显示负号，温湿度正常 |
| 温度上越界 | 50.1C / 40% | 01H | F5H | 28H | 当前温度显示 HH.H |
| 温度下越界 | -10.0C / 35% | 80H | 64H | 23H | 当前温度显示 LL.L |
| 湿度上越界 | 20.0C / 100% | 00H | C8H | 64H | 当前湿度显示 HH |
| 湿度下越界 | 20.0C / 0% | 00H | C8H | 00H | 当前湿度显示 LL |
| 干燥图标 | 20.0C / 29% | 00H | C8H | 1DH | Dry 亮 |
| 舒适图标 | 20.0C / 50% | 00H | C8H | 32H | Comfort 亮 |
| 潮湿图标 | 20.0C / 70% | 00H | C8H | 46H | Wet 亮 |

补充说明：

1. 温度大于 50.0C 才会 HH，所以 50.0C 本身是正常显示，50.1C 才是更稳妥的越界点。
2. 温度低于 -9.9C 才会 LL，所以用 -10.0C 最直接。
3. 湿度趋势阈值是超过 5%RH，建议至少用 6%RH 的跨度来测。
4. 温度趋势阈值是超过 1.0C，建议至少用 1.1C 的跨度来测。

## 5. 历史功能怎么测

### 5.1 Today / 48hr / All-time 基本切页

步骤：

1. 上电，等待界面稳定。
2. 保持标准页，确认当前页签就是 Today。
3. 短按 ALL-TIME 一次，应该进入历史页 48hr。
4. 再短按 ALL-TIME 一次，应该切到 All-time。
5. 再短按 ALL-TIME 一次，应该回到 48hr。
6. 停止操作，等待 20 秒。

预期：

1. 标准页显示 Today 页签。
2. 历史页只在 48hr 和 All-time 之间切换。
3. 20 秒无操作后自动回标准页。

### 5.2 清除范围隔离测试

这条是最关键的逻辑测试，推荐优先做。

先准备 3 个连续样本：

1. 样本 A：20.0C / 40%
2. 样本 B：30.0C / 60%
3. 样本 C：25.0C / 50%

把这 3 个样本按历史采样顺序写进去后，理论上：

1. Today 温度 max/min 应该是 30.0 / 20.0。
2. Today 湿度 max/min 应该是 60 / 40。
3. 当前值停在样本 C，也就是 25.0 / 50。

然后分 3 轮测试：

#### A. 在标准页长按 F/C/CLEAR

预期：

1. Today 前台先显示约 3 秒横杠。
2. 3 秒后 Today 记录应该被当前样本 C 重建，所以 Today max/min 变成 25.0 / 25.0，湿度变成 50 / 50。
3. 48hr 不应该被清掉，仍应保留原先 30.0 / 20.0 和 60 / 40。
4. All-time 也不应该被清掉。

#### B. 在 48hr 页长按 F/C/CLEAR

预期：

1. 48hr 前台先显示约 3 秒横杠。
2. 3 秒后 48hr 会用当前样本 C 重建，所以 48hr max/min 变成 25.0 / 25.0，湿度变成 50 / 50。
3. Today 不应变化。
4. All-time 不应变化。

#### C. 在 All-time 页长按 F/C/CLEAR

预期：

1. All-time 前台先显示约 3 秒横杠。
2. 3 秒后 All-time 只剩当前样本 C。
3. Today 不应变化。
4. 48hr 不应变化。

### 5.3 48hr 聚合测试

这条要验证 48hr 确实来自 48HrToday 和 Previous Day 的聚合，而不是独立单桶。

建议做法：

1. 用软件注数。
2. 再配合调试器把 RTC 改到跨日点附近，避免真的等到第二天。

推荐序列：

1. 前一天样本 A：20.0C / 40%
2. 前一天样本 B：28.0C / 60%
3. 把 RTC 调到 23:59:59，等待跨到 00:00:00
4. 当天样本 C：22.0C / 50%
5. 当天样本 D：31.0C / 75%

预期：

1. Today 应该只看当天，也就是温度 31.0 / 22.0，湿度 75 / 50。
2. 48hr 应该聚合昨天和今天，也就是温度 31.0 / 20.0，湿度 75 / 40。
3. All-time 如果没清过，结果会至少覆盖这两天全部样本。

如果 48hr 和 Today 一样，通常优先排查：

1. Previous Day 是否真的在 00:00:00 时被复制。
2. 当前是否仍停留在同一天。
3. 48hr 页是否真正走到了 HistoryLoadView_48Hr。

## 6. 其他功能怎么测

### 6.1 C/F 切换

步骤：

1. 在标准页短按 F/C/CLEAR。
2. 再去 48hr 和 All-time 看 max/min 温度显示。

预期：

1. 当前温度单位切换正常。
2. 历史 max/min 也跟着切单位。
3. 切单位不应改变历史记录本身，只是显示单位变了。

### 6.2 温湿舒适度图标

当前阈值：

1. 湿度小于 30 显示 Dry。
2. 湿度 30 到 69 显示 Comfort。
3. 湿度大于等于 70 显示 Wet。

建议直接用 29、50、70 三组湿度去打。

### 6.3 趋势图标

当前逻辑：

1. 温度窗口跨度大于 1.0C 时才触发方向图标。
2. 湿度窗口跨度大于 5%RH 时才触发方向图标。
3. 如果 60 次采样都没有触发阈值，显示平缓图标。

快验建议：

1. 温度 Up：先打 25.0C，再打 26.1C。
2. 温度 Down：再打 24.8C。
3. 湿度 Up：先打 50%，再打 56%。
4. 湿度 Down：再打 49%。

说明：

1. 平缓图标要等 60 次采样，按真实节拍就是 60 分钟。
2. 如果只是验证 Up/Down，软件注数已经够用。
3. 如果要快测平缓，建议调试器配合改时间基准或直接观察计数变量。

### 6.4 Mold 阈值设置和报警

步骤：

1. 在标准页长按 ALL-TIME 3 秒，进入 Mold 设置页。
2. 短按 F/C/CLEAR，阈值应在 60、65、70、75 之间循环。
3. 停止操作 10 秒，应自动回标准页。
4. 分别打低于阈值和高于等于阈值的湿度值。

预期：

1. 高于等于阈值时 Mold 图标亮。
2. 报警蜂鸣按既定节奏响。
3. 低于阈值时图标灭，蜂鸣停止。

建议样例：

1. 阈值设 65。
2. 打 64%，应不报警。
3. 打 65%，应报警。
4. 打 70%，应持续报警。

### 6.5 背光

步骤：

1. 短按 LIGHT。

预期：

1. 背光立即点亮。
2. 约 8 秒后熄灭。

### 6.6 温湿度越界显示

建议验证以下点：

1. 50.0C 正常显示，不应 HH。
2. 50.1C 显示 HH.H。
3. -9.9C 正常显示，不应 LL。
4. -10.0C 显示 LL.L。
5. 湿度 100% 当前值显示 HH。
6. 湿度 0% 当前值显示 LL。

补充：

1. 当前温度一旦 HH 或 LL，不再写入温度历史 max/min。
2. 但湿度历史仍然会继续更新，所以要顺手确认温度异常时湿度历史没有被一起打坏。

### 6.7 低电测试

仓库里已经有现成文档：LowBattery_TestFlow.md。

建议直接按里面的电压步骤做，因为这部分需要可调电源，纯软件注数不能替代。

### 6.8 页面超时返回

步骤：

1. 进入 48hr 或 All-time。
2. 不做任何操作，等待 20 秒。
3. 进入 Mold 设置页，不操作，等待 10 秒。

预期：

1. 历史页 20 秒后回标准页。
2. Mold 设置页 10 秒后回标准页。

## 7. 我建议你实际测试的顺序

为了省时间，按下面顺序跑最划算：

1. 先做页面切换和超时返回。
2. 再做 Today / 48hr / All-time 清除隔离。
3. 再做 48hr 跨日聚合。
4. 再做 C/F 切换、舒适图标、趋势图标。
5. 最后做 Mold、低电、背光和越界。
